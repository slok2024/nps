NPS 是一款使用 Go 语言编写的轻量级、高性能、功能强大的内网穿透代理服务器。它能够通过一个具有公网 IP 的服务器，将内网环境下的各种服务（如 Web 服务、数据库、远程桌面等）暴露到公网，方便远程访问。


多协议支持：
TCP/UDP 隧道：支持基本的端口转发。
HTTP/HTTPS 代理：支持域名解析，可以将不同的域名映射到不同的内网 Web 服务上。
SOCKS5 代理：可以作为内网穿透的安全代理通道。

Web 管理界面：
不同于传统的穿透工具（如 frp 需频繁修改配置文件），NPS 提供了一个非常直观的网页后台。
你可以实时查看客户端的连接状态、流量统计、配置隧道等，操作门槛更低。
多平台兼容：
内核支持 Windows、Linux、MacOS 以及各种 ARM 架构设备。
采用了 Server/Client (nps/npc) 架构，服务端负责流量中转，客户端负责内网连接。
安全性与性能：
支持验证密钥 (vkey) 校验，确保只有授权的客户端可以连接。
支持数据压缩和加密传输，保障数据安全的同时提高传输效率。

工作原理简述
服务端 (NPS) 运行在公网服务器，监听 桥接端口（如你设置的 8024）等待客户端连接。
客户端 (NPC) 运行在内网环境，主动连接服务端的桥接端口并维持长连接。
流量转发：当外网用户访问服务端映射的公网端口时，NPS 会将流量通过之前的长连接“泵入”内网，由 NPC 转发给最终的目标服务。
